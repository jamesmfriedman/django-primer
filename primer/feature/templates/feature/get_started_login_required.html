Login Required @public

Django defaults everything to being public and has you place an @login_required decorator on every view that requires login. I don't know about you, but 99.99% of my web app is login required, making typing this extremely ugly and redundant. Primer has an optional middlware you can include that flips this functionality and mixes it with an @public decorator.

Useage

This functionality is opt-in, so you have to include the middleware in your settings file.

MIDDLEWARE_CLASSES = (
	...
    'primer.middleware.LoginRequiredMiddleware',
)

And now, somewhere in a view...

from primer.auth import public

@public
def my_view(request):
	...

Setting Public Modules

If you are using a third party app with views, you can set the PUBLIC_MODULES list in your settings file. This will allow you to make the 3rd party app public, or save you the redundancy of including the decorator over and over again in your own views if the entire module is open. Django, and the appropriate parts of Primer are added for you automatically.

PUBLIC_MODULES = (
    'myapp.views',
    'core.apps.foo',
)
