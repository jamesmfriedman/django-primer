Settings

Primer comes with a quite a few apps that have a multitude of settings. In order to keep your settings file clean and pretty, the easiest thing to do is to include Primer's settings at the end of your settings file.

# this should be the last thing in your settings file
from primer.settings import *

What is it doing?

Since Primer is kind of a framework on a framework, importing the settings file does a bit of magic rewriting. It will include Primer's apps, middleware, and template context processors after Django's, but before your own. It will also add in any settings it needs that you didn't explicitly declare back into your setting's module's namespace.

Setting Default Description
APP_ROOT Current Path This will be the path that your application is running from.
COMPRESS_URL settings.STATIC_URL The compress url for django-compressor. Django Compressor
COMPRESS_ENABLED True if debug is off. Whether or not to compress and minify js and css. It is a good idea to turn this on every once in a while in development to make sure your JS and CSS work minified.
PUSH_SERVICE None Should be set to the name of a push service you want to use.This can be 'pubnub', 'pusher', or a string pointing to a custom class.
PUSH_SERVICE_SETTINGS {} A dictionary of settings for your push service. Read Get Started > Push Services for more info.
LESS_CSS_PATHS {} A dictionary of less inputs pointing to css outputs. See above example.
LESS_PROCESSOR_ENABLED value of settings.DEBUG Whether or not you want to use the less processor. Defaults to your settings.DEBUG value, basically saying it is on by default if debug is on.
LESS_ROOT settings.APP_ROOT This defaults to your applications root. You shouldn't have to change this, but I can't account for your DEV setup. This is the jumping off point for the lessc processor.
